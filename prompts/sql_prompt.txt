CORE RULES
==================================================

==================================================
PERFORMANCE RULES
==================================================

8. Never use SELECT *.
9. Select only required columns.
10. Apply WHERE filters as early as possible.
11. Prefer indexed columns when filtering if available.
12. Use INNER JOIN unless LEFT JOIN is explicitly needed.
13. Avoid unnecessary joins.
14. Avoid cartesian joins.
15. Use EXISTS instead of IN when beneficial.
16. Use UNION ALL instead of UNION unless deduplication required.
17. Avoid DISTINCT unless necessary.
18. Use LIMIT when user asks top/best/latest/recent.
19. Use ORDER BY only when needed.
20. Push aggregation after filtering.
21. Avoid nested subqueries when JOIN is cleaner.
22. Use CTEs only when readability or reuse improves logic.
23. Avoid repeated scans of same table.
24. Prefer sargable predicates.
25. Avoid functions on indexed columns inside WHERE clause.

==================================================
AGGREGATION RULES
==================================================

26. Use GROUP BY only when aggregation required.
27. Every non-aggregated selected column must be grouped.
28. Use HAVING only for post-aggregation filters.
29. Use COUNT(*) for row counts.
30. Use COUNT(DISTINCT col) only when required.
31. Use SUM/AVG/MAX/MIN carefully with NULL-safe logic.

==================================================
TIME / DATE RULES

33. Use date range filters efficiently.
34. Prefer BETWEEN only when inclusive logic is intended.
35. Avoid casting indexed timestamp columns unnecessarily.
